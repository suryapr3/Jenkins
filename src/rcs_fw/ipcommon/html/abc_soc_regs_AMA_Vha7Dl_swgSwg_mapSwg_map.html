+0x00000000[+=0x40][+=0x4] Register(32 bit) INIT_UPPER[2][4]
Upper 6 bits [37:32] of init_upper
This register contains the 6 MSbits of the 38-bit initial theta value for the given Cordic.
To calculate the initial values of the N Cordics, the user must first calculate Angle_between_cordics.
[code]
Angle_between_cordics (in degrees) = 360 * (f_out in GHz)/(f_effective sysclk f in GHz)

where:
fout is the desired output frequency of sine wave generator
f_effective_sysclk is the effective sysclk after multiplying sysclk by N = number of
active cordics (typically 8 or 16)

init_angle[Cordic_Number] (in degrees) = angle_between_cordics x [N - Cordic_Number -1]
where: cordic_number = 0, 1, 2,...N-1, where N = number of active cordics.
[/code]
To convert degrees to an internal 38-bit value, use the following formula
[code]
converted_angle[37:0] = (angle_in_degrees/360) * 2^38.
[/code]
Although the N Cordics generate their N outputs in parallel,
they are effectively used serially as though there was a clock operating
at a frequency = N x freq_sys_clk. As such, phase order of the outputs of N-phase Cordic
sine wave generator does matter. We want the earliest output of the N-Phase Cordic
generator to line up with the earliest sample in the data stream.
The earliest output of the N-Phase Cordic generator should be from Cordic[N-1].
The last output of the N-Phase Cordic generator should be from Cordic[0].
For example, if the DUC is supplying 16 samples to the Modulator and the NCO
comprises a 16-Phase Cordic Sine Wave Generator, then if we want the sine wave
generator's first output to be equal to 0, we would set cordic_init[15] to Zero degrees
and this value would be multiplied by the value of input_sample[15].
The antenna modulator's Sine Wave Generator generates a single sinewave with N samples per clock cycle when operating
in NGSPS mode.

+0x000004a8 Register(32 bit) ENABLE_PER_SWG
bitwise enable for all the CORDICs in the given SWG
Each instantiation of a SWG instantiates N_SWG independent sine wave registers.
This register allows the user to independently enable or disable each
individual sine wave generator by setting the appropriate bit in
bit position (N_SWG-1):0.
1: enables the corresponding sinewave generator.
0: disables the corresponding sine wave generator.
When SWG is instantiated in FB_SYS, N_SWG=6 .
The corresponding bit positions are as follows:
0: one NCO for RXFB0 demodulator receiving data from 4 antenna channels 0-3
1: one NCO for RXFB1 demodulator receiving data from 4 antenna channels 4-7
2: one NCO for all Rx (DMOD) channels 0-3
3: one NCO for all Tx (MOD) channels 0-3
4: one NCO for all Rx (DMOD) channels 4-7
5: one NCO for all Tx (MOD) channels 4-7
When SWG is instantiated in VHA, N_SWG=4
0: one NCO for modulator channel band 0
1: one NCO for modulator channel band 1
2: one NCO for demodulator channel band 0
3: one NCO for demodulator channel band 1
When SWG is instantiated in the CALDAC, N_SWG = 2
0: one NCO for channel 0
1: one NCO for channel 1

+0x000004ac Register(32 bit) EXT_BYP
allows the external trigger enable to be bypassed
The user can enable the SWG directly by setting the correspoding bits in the sgw_enable_per_swg register and then asserting the corresponding byp_ext bit. Normally, the SWG is enabled using a trigger external to the SWG. When using the external trigger, the SWG will be enabled as long as that trigger input remains high. Some modules using the SWG use bfn events as the trigger. Other modules use the FIFO almost full signal as the trigger. The trigger source is not limited to these 2 examples .

dual_fcw_enable

When set to 0, the SWG only used FCW0 (frequency control word 0, or incr) when generating its output. When set to 1, the SWG is configured to use 2 FCW (frequency control word, also referred to as incr) to control the frequency of the SWG output. In this mode of operation, FCW0 (incr) is used for M clock cycles and FCW1 (incr1) is used for N clock cycles every M+N clock cycles. Hence, the SWG uses the following control word sequence: FCW0, FCW1, FCW0, FCW1, ....
