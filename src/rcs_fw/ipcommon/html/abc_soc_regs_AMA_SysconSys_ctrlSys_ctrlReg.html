+0x00000008 Register(32 bit) System_Control
System Control Register
SYSCON System Control Register. Clock mux control. Warm and cold reset generation.

Sequence to initiate warm reset:
1. Write 0x4XXXXXXX to System Control Register - enables warm reset
2. Write 0x5XXXXXXX to System Control Register - generates warm reset
3. Reset will clear these bits.

Sequence to initiate cold reset:
1. Write 0x8XXXXXXX to System Control Register - enables cold reset
2. Write 0xAXXXXXXX to System Control Register - generates cold reset
3. Reset will clear these bits.

Cold reset always implies warm reset (in HW), but SW cannot
initiate both (0xCXXXXXXX, followed by 0xFXXXXXXX will not do
anything because the SWRst_EN is not correct for either cold or warm reset.
Writing 0x1XXXXXXX, 0x2XXXXXXX and 0x3XXXXXXX also will do nothing (SWRst_EN = 2'b00).
Finally, setting both SW*_Rst bits will do nothing, so
0x7XXXXXXX, 0xBXXXXXXX will do nothing, regardless the state of SWRst_EN.

Note, the sequence: 0x6XXXXXXX, 0x5XXXXXXX will also generate a warm reset and
the sequence: 0x9XXXXXXX, 0xAXXXXXXX will generate a cold reset

Function table for {SWRst_EN[1:0],SWCold_Rst,SWWarm_Rst}:

0x0: NO-OP, SWRst_EN = 2'b00
0x1: NO-OP, SWRst_EN = 2'b00
0x2: NO-OP, SWRst_EN = 2'b00
0x3: NO-OP, SWRst_EN = 2'b00
0x4: Enable warm reset
0x5: Generate warm reset
0x6: Enable warm reset (SWCold_Rst ignored)
0x7: NO-OP, SWCold_Rst and SWWarm_Rst both set
0x8: Enable cold reset
0x9: Enable cold reset (SWWarm_Rst ignored)
0xA: Generate cold reset
0xB: NO-OP, SWCold_Rst and SWWarm_Rst both set
0xC: NO-OP, SWRst_EN = 2'b11
0xD: NO-OP, SWRst_EN = 2'b11
0xE: NO-OP, SWRst_EN = 2'b11
0xF: NO-OP, SWRst_EN = 2'b11

+0x00000010 Register(32 bit) Reset_Control
SYSCON Reset Control Register
Global Reset Control. Used by SW to release subfc and local resets after boot.
All bits represent ACTIVE LOW resets, e.g. 0=reset active.
SW can write a '1' to any '0' bit to release the block from reset.
Some bits are set to '1' by HW during boot.

For Functional (non-Coresight) resets: Writing a '0' will have no effect: SW CANNOT initiate individual block resets.

For Coresight resets: Writing a '0' will disable the Coresight clocks and then assert the reset to the Coresight SS
Writing a '1' wiill release the Coresight SS from reset and re-enable the Coresight clocks.
