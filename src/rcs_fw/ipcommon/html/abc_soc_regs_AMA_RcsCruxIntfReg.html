+0x00000090 Register(128 bit) CRUX_RX_POP
POP rx data from Common module fifo.
When this register is read, it will cause a pop of the B2H (rx) fifo in the nsip_hbi. It is set to do this on read to avoid the situation where a pop cant be done because writes are blocked on AXI interface due to a full TX fifo. This register will always read back as 0. The contents will be stored in CRUX_RDATA0-3, CRUX_RROUTING_DATA. Poping when the B2H fifo is empty will cause an error.
