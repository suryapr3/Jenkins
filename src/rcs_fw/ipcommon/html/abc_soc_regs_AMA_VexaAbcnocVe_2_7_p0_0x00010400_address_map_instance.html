+0x00000038 Register(64 bit) bridge_ve_2_7_p0_23_58_noc_tx_qos_weight_0_value
QoS profile data
This register describes the weight value of each QoS supported at the noc_tx. Each byte of this register must be greater than or equal to 1. Each transmitting noc_tx supports up to 16 QoS profiles. Each QoS is composed of priority and weight, however only the weight is programmable, therefore is part of the registers.QoS data is composed of four registers, noc_tx_qos_weight_0, noc_tx_qos_weight_1, noc_tx_qos_weight_2 and noc_tx_qos_weight_3, each of which contains the weight of four profiles. Depending upon how many QoS profiles are enabled, the appropriate bits in the following registers are available.

Interrupt status register
This register tracks error or interrupt conditions. It resets to 0, but as these conditions occur, the corresponding bits are set to 1. This register can be read and can also be cleared by sending a write with bits set to 0 for the bits that should be cleared. This register works in combination with the Transmit Interrupt Mask register to determine when an interrupt is transmitted.
